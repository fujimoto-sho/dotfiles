# Commit Command

## 実行順序

1. **Format実行（必須）**
   - `pnpm format` を実行
   - 失敗した場合 → 処理を中断し、エラー内容を報告
   - 成功した場合 → 次へ進む

2. **Type Check実行（必須）**
   - `pnpm type-check` を実行
   - 失敗した場合 → 処理を中断し、エラー内容を報告
   - 成功した場合 → 次へ進む

3. **Lint実行（必須）**
   - `pnpm lint` を実行
   - 失敗した場合 → 処理を中断し、エラー内容を報告
   - 成功した場合 → 次へ進む

4. **Test実行（必須）**
   - `pnpm test` を実行
   - 失敗した場合 → 処理を中断し、エラー内容を報告
   - 成功した場合 → 次へ進む

5. **変更確認**
   - `git status` で変更ファイルを確認
   - `git diff` で変更内容の詳細を確認

6. **ステージング**
   - `git add .` ですべての変更を追加

7. **コミット実行**
   - 変更内容を分析してコミットメッセージを生成
   - `git commit -m "生成したメッセージ"` を実行

## 制約事項

- ✅ format / type-check / lint / test がすべて成功した場合のみコミット可能
- ❌ `git push` は実行しない（プッシュ禁止）
