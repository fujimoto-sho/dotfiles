git diffを実行し、以下の入力（diff/コード）をレビューしてください。

# 出力ルール
- 必ず日本語
- 重要度順（P0/P1/P2/P3）
- 可能なら「最小修正案」を添える
- 問題がなければ最後に **LGTM** のみでOK（ただしP2以上があればLGTM禁止）

# チェック観点
- ロジックの正しさ / エッジケース
- パフォーマンス（無駄な再計算、N+1、無限ループ等）
- セキュリティ（権限漏れ、入力検証、情報漏洩）
- 命名/責務/可読性
- 破壊的変更の影響範囲
